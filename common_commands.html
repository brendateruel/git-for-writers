Common Commands

Here is a list of common commands that may come in handy for writers

Setting up your user name

One of the first things you'll want to do after installing Git is set up your username and email address. Every time you commit a change, Git will automatically sign your change with your username and email address. So, be sure to use a name and email address that you want others to see when looking at your Git repository. 

$ git config --global user.name "Your name"
$ git config --global user.email "your_email_address@example.com"

Initializing a New Repository

To start tracking project files with Git, you'll want to select or create a new folder to house the repository files (ex: my-project). Once you've decided the directory, go to it in Git and initialize a new repository. 

$ cd my-project
$ git init

Staging a Commit

In Git, we stage a file first before committing. We can check which files might be ready for staging by running a status check. But, Git only sees changes to files it is already tracking so we may need to add files to the working directory first. 

$ git status
# On branch master
# Untracked files:
#   (use "git add ..." to include in what will be committed)
#
#	new.txt
nothing added to commit but untracked files present (use "git add" to track)

In the example above, new.txt is not being tracked yet and still needs to be added to the working directory. Once we perform the add command, the file will be staged for a commit. (You can run a status check again to be sure of this.) However, if we modify it again before committing, we'll need to restage it once more. 

$ git add new.txt
$ git status
# On branch master
# Changes to be committed:
#   (use "git reset HEAD ..." to unstage)
#
#	new file:   new.txt
#

Making a Commit

Once one or more files are staged, we can commit the changes. To add details about the commit, we can add "-m" + "[commit details go here]" to the end of the command. 

$ git commit -m "updated 3 files"

We can also add/stage and commit a file at the same time by adding "-a" to the command. 

$ git commit -a -m "added new"
